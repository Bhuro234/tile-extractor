Native XREF Engine.

By parsing the exact binary dictionary of the PDF file format, the native engine extracts embedded image XObjects without any re-encoding. The result is 100% original, bit-for-bit lossless quality.

for xref in doc.get_page_images(page):
    base_image = doc.extract_image(xref)
    image_bytes = base_image["image"]

Optical Vision Engine.

Built for the complex layouts. When pages are flattened or scanned, advanced computer vision takes over. It rasterizes the page, applies strict binary thresholding, and utilizes contour detection to identify mathematical bounding boxes around physical tiles.

gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
_, thresh = cv2.threshold(gray, 240, 255, ...)
contours, _ = cv2.findContours(thresh, ...)
OpenCV NumPy Thresholding

Smart Strip Slicing.

Intelligent aspect ratio analysis. When tiles are placed closely together in catalogues, they form monolithic blocks. The engine calculates pixel intensity columns across the block, pinpointing the microscopic white gaps to mathematically slice the block back into independent squares.

Vertical Projection Micro-Gap Detection

Deduplication Engine.

Because the dual-engine scans aggressively, it uses MD5 cryptographic hashing on the raw byte stream of every extracted tile. Duplicates are instantly identified and destroyed, keeping the final output completely unique.

MD5 Hash Memory Registry

Precision Filtering.

The engine implements strict algorithmic filters to ensure only true tiles make it to the gallery.

Bi-tonal Gap & Shard Prevention

By analyzing column standard deviation and mean intensities, the Strip Slicer detects both pure white and pure black gaps. A >100px proximity lock prevents shattering tiles into micro-shards.

Dimensional Noise Rejection

The OCR limits bounds mathematically. Images under 150x150 pixels (social icons, bullet points) are destroyed, and bounding boxes exceeding 50% of the page area (full-page backgrounds) are instantly rejected.

Spatial Data Harvesting (v4).

An advanced subsystem built to scrape catalogue specifications by mapping text coordinates directly to extracted tile images.

Bounding-Box Matching

Extracts the exact (x, y) pixel coordinates of every text block on the page. By geometrically calculating the distance between text and image crops, the engine explicitly binds product names to their physical tiles.

Regex Attribute Parsing

Scans bound text using strict Regular Expressions to automatically detect dimensions (e.g., 600x1200mm), thickness (e.g., 9.5 mm), and cross-references surface finishes against a known dictionary.
